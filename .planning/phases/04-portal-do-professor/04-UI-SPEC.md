## Spacing Scale

Declared values (Tailwind spacing scale, multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (`gap-1`, `px-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`, `space-y-2`) |
| md | 12px | Button compact padding (`px-3 py-1`) |
| lg | 16px | Default element spacing (`gap-4`, `p-4`, `px-4 py-3`) |
| xl | 24px | Section padding (`p-6`) |
| 2xl | 32px | Layout gaps (`gap-8`, `mb-8`) |
| 3xl | 48px | Page-level spacing (`p-8`, `mb-12`) |

Exceptions:
- Page top padding: `pt-16` to clear fixed AppLayout header (h-16)
- Modal max-height: `max-h-[90vh]` (not a multiple of 4, required for viewport fitting)
- Sidebar width: `w-56` (224px, admin only — professor portal does NOT use sidebar)
- Badge micro-padding: `py-0.5` (2px, compact alert badge — matches existing EntityTable badge pattern)

## Color

| Role | Value | Tailwind Class | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | white | `bg-white` | Page backgrounds, modal surfaces, cards |
| Secondary (30%) | gray-50 | `bg-gray-50` | Table headers, subtle card backgrounds, AppLayout page bg |
| Accent (10%) | indigo-600 | `bg-indigo-600`, `text-indigo-600`, `focus:ring-indigo-500` | Primary buttons, active nav states, links, focus rings |
| Destructive | red-600 | `bg-red-600`, `text-red-600`, `hover:bg-red-700` | Danger buttons, destructive actions, error links |

Accent reserved for:
- Primary action buttons (`bg-indigo-600 hover:bg-indigo-700`)
- Active navigation links (`bg-indigo-600 text-white`)
- Interactive focus states (`focus:ring-2 focus:ring-indigo-500`)
- Hyperlinks (`text-indigo-600 hover:text-indigo-800`)

Additional semantic colors:

| Semantic | Background | Text | Usage |
|----------|------------|------|-------|
| Success | `bg-green-100` | `text-green-800` | Status badges (Ativo, Presente) |
| Danger badge | `bg-red-100` | `text-red-800` | Status badges (Inativo, Falta) |
| Error banner | `bg-red-50` | `text-red-700` | Form validation errors, API error messages |
| Warning | `bg-yellow-50` | `text-yellow-800` | Alert badges (frequencia baixa) |
| Muted | — | `text-gray-400` | Loading text, empty states, placeholders |

## Layout Patterns

### Professor Portal Structure
- Uses `AppLayout` (top header, NOT sidebar-based like admin)
- Header: fixed top, `h-16`, `bg-white shadow-sm border-b border-gray-200`
- Content: `pt-16` offset, full-width with `max-w-7xl mx-auto px-4 sm:px-6 lg:px-8`
- On mobile (`< 640px`): padding reduces to `px-4`

### Modal Pattern (reused from admin)
- `createPortal` to document.body
- Backdrop: `fixed inset-0 z-50 bg-black/40 p-4`
- Container: `bg-white rounded-lg shadow-xl w-full max-w-lg max-h-[90vh] overflow-y-auto`
- Header: `px-6 py-4 border-b border-gray-200` with close button (`text-gray-400 hover:text-gray-600`)
- Body: `px-6 py-4`
- Close triggers: ESC key, backdrop click, x button

### Table Pattern (reused from admin)
- Wrapper: `overflow-x-auto`
- Table: `min-w-full divide-y divide-gray-200`
- Header row: `bg-gray-50`
- Header cell: `px-4 py-3 text-left text-xs font-medium text-gray-500 uppercase tracking-wider`
- Body row: `hover:bg-gray-50`
- Body cell: `px-4 py-3 text-sm text-gray-700 whitespace-nowrap`

## Screen Contracts

### 1. Professor Landing Page (`/professor`)

**Layout:**
- Page heading: `text-2xl font-bold text-gray-900` — "Minhas Turmas"
- Subheading: `text-sm text-gray-500` — "Selecione uma turma para registrar chamada ou lancar notas"
- Cards grid: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4 mt-6`

**TurmaCard:**
- Container: `bg-white rounded-lg border border-gray-200 p-4 hover:border-indigo-300 hover:shadow-md transition-all cursor-pointer`
- Title: `text-xl font-semibold text-gray-900` — nome da turma
- Disciplina(s): `text-sm text-gray-500` — comma-separated disciplina names
- Alunos count: `text-sm text-gray-700` with icon — "X alunos"
- Active state (hover): `border-indigo-300 shadow-md`

**Empty state:**
- Icon: inline SVG (book-open or users) in `text-gray-300` at 48px
- Heading: `text-xl font-medium text-gray-900` — "Nenhuma turma vinculada"
- Body: `text-sm text-gray-500` — "Entre em contato com a administracao para vincular turmas ao seu perfil."

**Loading state:**
- `text-gray-400 text-sm` — "Carregando turmas..."

### 3. Chamada Tab

**Date selector:**
- Label: `text-sm font-medium text-gray-700 mb-1` — "Data da chamada"
- Input: `<input type="date">` with `border border-gray-300 rounded-md text-sm px-3 py-2 focus:outline-none focus:ring-2 focus:ring-indigo-500`
- Default value: today's date (`new Date().toISOString().split('T')[0]`)

**Existing record warning:**
- Banner: `bg-yellow-50 border border-yellow-200 rounded-md p-3 mb-4`
- Text: `text-sm text-yellow-800` — "⚠ Chamada ja registrada nesta data. Editar ira sobrescrever os registros anteriores."

**Student list:**
- Table or list layout (list preferred for mobile):
  - Container: `divide-y divide-gray-200 border border-gray-200 rounded-lg overflow-hidden`
  - Row: `flex items-center justify-between px-4 py-3 bg-white hover:bg-gray-50`
  - Student name: `text-sm font-medium text-gray-900`
  - Toggle: two-button group

**AttendanceToggle:**
- Container: `flex rounded-md shadow-sm`
- "Presente" button: `px-3 py-1 text-xs font-medium rounded-l-md border` — active: `bg-green-100 text-green-800 border-green-200`, inactive: `bg-white text-gray-500 border-gray-300 hover:bg-gray-50`
- "Falta" button: `px-3 py-1 text-xs font-medium rounded-r-md border -ml-px` — active: `bg-red-100 text-red-800 border-red-200`, inactive: `bg-white text-gray-500 border-gray-300 hover:bg-gray-50`
- Default state: **Presente** selected for all students

**Save action:**
- Button: `px-4 py-2 text-sm font-medium text-white bg-indigo-600 border border-transparent rounded-md hover:bg-indigo-700 disabled:opacity-50`
- Label: "Salvar chamada"
- Loading label: "Salvando..."
- Position: right-aligned below list, `mt-4 flex justify-end`

**Toast feedback (sonner):**
- Success: "Chamada salva com sucesso."
- Error: "Erro ao salvar chamada. Tente novamente."

**Empty state (no students in turma):**
- `text-center py-12 text-gray-400 text-sm` — "Nenhum aluno matriculado nesta turma."

### 4. Notas Tab

**Context bar:**
- Shows: disciplina name (if single) or disciplina selector
- If multiple disciplinas for this turma: dropdown select `border border-gray-300 rounded-md text-sm px-3 py-2`

**GradeTable:**
- Table: `min-w-full divide-y divide-gray-200`
- Columns: Aluno | 1o Bimestre | 2o Bimestre | 3o Bimestre | 4o Bimestre
- Header: `bg-gray-50` with `text-xs font-medium text-gray-500 uppercase tracking-wider`
- Rows: `hover:bg-gray-50`
- Student name cell: `px-4 py-3 text-sm font-medium text-gray-900`
- Input cell: `px-2 py-2`
- Input: `w-20 px-2 py-1 border border-gray-300 rounded-md text-sm text-center focus:outline-none focus:ring-2 focus:ring-indigo-500`
- Invalid input: `border-red-500 focus:ring-red-500` + `bg-red-50`
- Input type: `text` (to allow empty) with numeric validation; placeholder: "—"

**Validation rules (visual only):**
- Red border (`border-red-500`) when value is not a number or is outside 0–10 range
- Invalid state prevents save (button disabled until all valid)

**Keyboard navigation:**
- Tab: moves to next cell (left-to-right, top-to-bottom)
- Enter: moves to next row, same column

**Save action:**
- Button: same primary style as Chamada
- Label: "Salvar notas"
- Loading label: "Salvando..."
- Position: `mt-4 flex justify-end`

**Toast feedback:**
- Success: "Notas salvas com sucesso."
- Error: "Erro ao salvar notas. Verifique os valores e tente novamente."

---

### 5. Frequencia Tab

**FrequencyTable:**
- Table: `min-w-full divide-y divide-gray-200`
- Columns: Aluno | Presenca | Aulas
- Header: `bg-gray-50` with standard table header classes
- Row: `hover:bg-gray-50`
- Student name: `text-sm font-medium text-gray-900`
- Percentage: `text-sm font-medium`
  - >= 75%: `text-gray-700`
  - < 75%: `text-red-700`
- Count: `text-sm text-gray-500` — "17/20 aulas"
- Alert badge (when < 75%): `inline-flex items-center px-2 py-0.5 rounded text-xs font-medium bg-red-100 text-red-800`
  - Content: "⚠ Abaixo de 75%"

**Highlight row for at-risk students:**
- Row background: `bg-red-50` for students below 75%
- This is IN ADDITION to the badge, ensuring visual accessibility

**Empty state:**
- `text-center py-12 text-gray-400 text-sm` — "Nenhuma chamada registrada para esta turma."

**Info banner (when data exists):**
- `bg-blue-50 border border-blue-200 rounded-md p-3 mb-4`
- Text: `text-sm text-blue-800` — "Alunos com frequencia abaixo de 75% estao em risco de reprovacao por falta (LDB art. 24, VI)."

## Responsive Behavior

### Breakpoints
- Mobile: `< 640px` (Tailwind `sm:`)
- Tablet: `640px – 1024px` (Tailwind `md:`, `lg:`)
- Desktop: `> 1024px`

### Mobile adaptations (`< 640px`)
- **Landing page cards**: `grid-cols-1` (full-width cards), `gap-4`, reduced padding `p-4`
- **Turma detail tabs**: horizontal scroll if needed (`overflow-x-auto` on tab container), tab text may truncate
- **Chamada list**: full-width rows, toggle buttons slightly smaller (`px-2 py-1`)
- **Notas table**: horizontal scroll (`overflow-x-auto`) — table is too wide for mobile
- **Frequencia table**: horizontal scroll (`overflow-x-auto`)
- **Page padding**: `px-4` instead of `p-8`
- **Save button**: full-width (`w-full`) on mobile, right-aligned on desktop

### Tablet adaptations (`640px – 1024px`)
- **Landing page cards**: `sm:grid-cols-2`
- **Page padding**: `sm:px-6`

## Accessibility

- All toggles are `<button>` elements with `aria-pressed` reflecting state
- Tab navigation uses `role="tablist"`, `role="tab"`, `role="tabpanel"`
- Date input uses native `<input type="date">` (accessible by default)
- Grade inputs use `aria-invalid="true"` when validation fails
- Color is never the sole indicator:
  - Attendance: text label + background color
  - Low frequency: badge text + red row background
  - Validation: red border + `aria-invalid`
- Focus visible: `focus:ring-2 focus:ring-indigo-500 focus:outline-none` on all interactive elements
