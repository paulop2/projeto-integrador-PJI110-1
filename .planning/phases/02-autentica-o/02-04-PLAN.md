- Password field: label "Senha", type toggled by useState (password/text), right-side eye icon button (use Unicode eye: 👁 is too decorative — use SVG or simple text "Mostrar"/"Ocultar" toggle link), validates on blur (required)

- Submit button: full-width, label "Entrar", disabled + shows "Carregando..." text (not SVG spinner — keep it simple) during mutation.isPending
- Error area: below submit button, shows inline error message in red text when mutation fails

State:
- `email`, `senha`: form field values
- `showPassword: boolean` for toggle
- `emailError`, `senhaError`: per-field blur validation errors
- `loginError`: API error message

Mutation (TanStack Query useMutation):
```typescript
const mutation = useMutation({
  mutationFn: (body: { email: string; senha: string }) =>
    api.post('/auth/login', body).then((r) => r.data),
  onSuccess: (data) => {
    login(data.access_token, data.user)
    navigate(`/${data.user.tipo}`, { replace: true })
  },
  onError: () => {
    setLoginError('Email ou senha incorretos')
  },
})
```

Import `api` from services/api.

- Email: check `!email.trim()` → "Email é obrigatório"; check basic format (includes '@') → "Email inválido"
- Senha: check `!senha.trim()` → "Senha é obrigatória"

  },

})
