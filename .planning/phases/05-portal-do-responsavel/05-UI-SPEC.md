## Spacing Scale

Declared values (Tailwind spacing scale, multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (`gap-1`, `px-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`, `space-y-2`) |
| lg | 16px | Default element spacing (`gap-4`, `p-4`, `px-4 py-3`) |
| xl | 24px | Section padding (`p-6`) |
| 2xl | 32px | Layout gaps (`gap-8`, `mb-8`) |
| 3xl | 48px | Page-level spacing (`p-8`, `mb-12`) |

Exceptions:
- Page top padding: `pt-16` to clear fixed AppLayout header (h-16)
- Modal max-height: `max-h-[90vh]` (not a multiple of 4, required for viewport fitting)
- Badge micro-padding: `py-0.5` (2px, compact status badge) — micro-exception for badge compactness, not part of standard scale
- Table cell compact: `px-3 py-2` (for dense boletim table with many columns)
- Compact button padding: `px-3 py-1` (12px horizontal, 4px vertical) — not part of declared scale, inherited pattern from Phase 4

## Color

| Role | Value | Tailwind Class | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | white | `bg-white` | Page backgrounds, modal surfaces, cards |
| Secondary (30%) | gray-50 | `bg-gray-50` | Table headers, subtle card backgrounds, AppLayout page bg |
| Accent (10%) | indigo-600 | `bg-indigo-600`, `text-indigo-600`, `focus:ring-indigo-500` | Primary buttons, active nav states, links, focus rings |
| Destructive | red-600 | `bg-red-600`, `text-red-600`, `hover:bg-red-700` | Danger buttons, destructive actions, error links |

Accent reserved for:
- Primary action buttons (`bg-indigo-600 hover:bg-indigo-700`)
- Active navigation links (`bg-indigo-600 text-white`)
- Interactive focus states (`focus:ring-2 focus:ring-indigo-500`)
- Hyperlinks (`text-indigo-600 hover:text-indigo-800`)
- Child selector active state

Additional semantic colors:

| Semantic | Background | Text | Usage |
|----------|------------|------|-------|
| Success | `bg-green-100` | `text-green-800` | Status badge "Aprovado", summary card "aprovado em todas" |
| Danger badge | `bg-red-100` | `text-red-800` | Status badge "Reprovado", summary card "em risco" |
| Danger row | `bg-red-50` | — | Table row highlight when frequência < 75% |
| Danger text | — | `text-red-700` | Frequência percentual below 75% |
| Warning | `bg-yellow-50` | `text-yellow-800` | Summary card warning state (border only) |
| Error banner | `bg-red-50` | `text-red-700` | API error messages |
| Muted | — | `text-gray-400` | Loading text, empty states, placeholders, dash for missing nota |

## Layout Patterns

### Responsável Portal Structure
- Uses `AppLayout` (top header, NOT sidebar-based like admin)
- Header: fixed top, `h-16`, `bg-white shadow-sm border-b border-gray-200`
- Content: `pt-16` offset, full-width with `max-w-7xl mx-auto px-4 sm:px-6 lg:px-8`
- On mobile (`< 640px`): padding reduces to `px-4`
- **Single page**: `/responsavel` — no nested routes needed (read-only, single resource)

### Card Pattern (SummaryCard)
- Container: `bg-white rounded-lg border shadow-sm p-6`
- Success variant: `border-green-200 bg-green-50`
- Warning variant: `border-yellow-200 bg-yellow-50`
- Danger variant: `border-red-200 bg-red-50`
- Icon + title row: `flex items-center gap-3 mb-2`
- Big number: `text-2xl font-semibold`
- Description: `text-sm`

### Table Pattern (BoletimTable)
- Wrapper: `overflow-x-auto` (horizontal scroll on mobile)
- Table: `min-w-full divide-y divide-gray-200`
- Header row: `bg-gray-50`
- Header cell: `px-3 py-2 text-left text-xs font-semibold text-gray-500 uppercase tracking-wider`
- Body row: standard `hover:bg-gray-50`; at-risk `bg-red-50 hover:bg-red-100`
- Body cell: `px-3 py-2 text-sm text-gray-700 whitespace-nowrap`
- Numeric cells: `text-center`

## Screen Contracts

### 1. Responsável Boletim Page (`/responsavel`)

**Page flow:**
- Post-login redirect lands here directly (natural path — only resource)
- Responsável with single child: page loads boletim immediately
- Responsável with multiple children: `ChildSelector` shown; first child auto-selected on load

---

#### 1.1 Child Selector (conditional)

**Visibility:** Only rendered when `filhos.length > 1`

**Layout:**
- Container: `mb-6`
- Label: `text-sm font-normal text-gray-700 mb-1` — "Filho(a)"
- Select: `w-full sm:w-72 border border-gray-300 rounded-md text-sm px-3 py-2 focus:outline-none focus:ring-2 focus:ring-indigo-500 bg-white`
- Options: child name (e.g., "João Silva — 7º Ano A")

**Behavior:**
- Changing selection triggers new API fetch for boletim data
- Loading state: select disabled, table shows skeleton or "Carregando..."

---

#### 1.2 Summary Card

**Position:** Below child selector (or at top if only one child), `mb-6`

**States:**

**A) All approved — no risk:**
- Container: `bg-green-50 border border-green-200 rounded-lg p-6`
- Icon: inline SVG check-circle in `text-green-600` at 24px
- Title: `text-xl font-semibold text-green-800` — "Aprovado em todas as disciplinas"
- Body: `text-sm text-green-700` — "Média e frequência estão dentro dos limites exigidos."

**B) At-risk disciplines exist:**
- Container: `bg-yellow-50 border border-yellow-200 rounded-lg p-6`
- Icon: inline SVG alert-triangle in `text-yellow-600` at 24px
- Title: `text-xl font-semibold text-yellow-800` — "X disciplina(s) em risco de reprovação"
- Body: `text-sm text-yellow-700` — "Verifique as disciplinas destacadas abaixo. Frequência abaixo de 75% ou média inferior a 5,0."

**C) Failed disciplines exist (all bimesters complete or definitive):**
- Container: `bg-red-50 border border-red-200 rounded-lg p-6`
- Icon: inline SVG x-circle in `text-red-600` at 24px
- Title: `text-xl font-semibold text-red-800` — "X disciplina(s) com reprovação confirmada"
- Body: `text-sm text-red-700` — "Média ou frequência não atingiram o mínimo exigido pela LDB."

> **Note:** The summary card uses the same approval rule as the row badges: `média >= 5.0 AND frequência >= 75%`. A discipline is "at risk" if either condition is not met.

---

#### 1.3 Boletim Table

**Heading:**
- Page title: `text-2xl font-semibold text-gray-900 mb-2` — "Boletim Escolar"
- Subtitle: `text-sm text-gray-500 mb-6` — "Ano letivo 2026" (or current year)

**Table structure:**

| Column | Alignment | Width | Content |
|--------|-----------|-------|---------|
| Disciplina | left | auto | Discipline name |
| 1º Bim. | center | 80px | Nota or "—" |
| 2º Bim. | center | 80px | Nota or "—" |
| 3º Bim. | center | 80px | Nota or "—" |
| 4º Bim. | center | 80px | Nota or "—" |
| Média | center | 80px | Calculated average (semibold) |
| Frequência | center | 120px | "82% (15/18)" |
| Status | center | 100px | Badge: Aprovado / Reprovado |

**Header row:**
- `bg-gray-50`
- Cell classes: `px-3 py-2 text-left (or text-center) text-xs font-semibold text-gray-500 uppercase tracking-wider`

**Data row:**
- Base: `hover:bg-gray-50`
- At-risk (frequência < 75% OR média < 5.0): `bg-red-50 hover:bg-red-100`
- Disciplina name: `text-sm font-semibold text-gray-900`
- Nota cells: `text-sm text-gray-700 text-center`
  - Missing nota: `text-gray-400` — "—" (em dash)
  - Nota value: `text-sm text-gray-700`
- Média cell: `text-sm font-semibold text-gray-900 text-center`
  - If média < 5.0: `text-red-700 font-semibold`
- Frequência cell: `text-sm text-center`
  - >= 75%: `text-gray-700`
  - < 75%: `text-red-700 font-semibold`
- Status badge cell: `text-center`

**StatusBadge:**
- "Aprovado": `inline-flex items-center px-2 py-0.5 rounded text-xs font-semibold bg-green-100 text-green-800`
- "Reprovado": `inline-flex items-center px-2 py-0.5 rounded text-xs font-semibold bg-red-100 text-red-800`

**Row highlight rule:**
- Row gets `bg-red-50` when **either** `frequência < 75%` **or** `média < 5.0`
- This ensures the parent spots problems immediately without reading every cell

**Footer / note below table:**
- `text-xs text-gray-500 mt-4`
- Content: "Média calculada automaticamente. Aprovação exige média ≥ 5,0 e frequência ≥ 75% (LDB art. 24, VI)."

---

#### 1.4 Loading State

**Skeleton / loading:**
- Summary card: gray placeholder bar (`bg-gray-200 rounded h-20 animate-pulse`)
- Table: 6 rows of pulsing gray bars mimicking the table structure
- Text: `text-gray-400 text-sm` — "Carregando boletim..."

---

#### 1.5 Empty States

**No children linked:**
- Icon: inline SVG users in `text-gray-300` at 48px
- Heading: `text-xl font-semibold text-gray-900` — "Nenhum aluno vinculado"
- Body: `text-sm text-gray-500` — "Entre em contato com a administração para vincular seu perfil a um aluno."

**No boletim data for selected child:**
- Icon: inline SVG file-text in `text-gray-300` at 48px
- Heading: `text-xl font-semibold text-gray-900` — "Boletim não disponível"
- Body: `text-sm text-gray-500` — "As notas e frequências ainda não foram lançadas para este aluno."

## Responsive Behavior

### Breakpoints
- Mobile: `< 640px` (Tailwind `sm:`)
- Tablet: `640px – 1024px` (Tailwind `md:`, `lg:`)
- Desktop: `> 1024px`

### Mobile adaptations (`< 640px`)
- **Child selector**: full-width (`w-full`)
- **Summary card**: full-width, padding `p-4`, icon + text stacked vertically if needed
- **Boletim table**: horizontal scroll mandatory (`overflow-x-auto`)
  - Table min-width: `min-w-[640px]` to force scroll
  - Reduce cell padding to `px-2 py-2`
  - Column widths shrink: bimester columns 60px, média 60px, frequência 100px, status 80px
- **Page padding**: `px-4`

### Tablet adaptations (`640px – 1024px`)
- **Page padding**: `sm:px-6`
- **Boletim table**: fits without scroll in most cases; `overflow-x-auto` remains as safeguard
- **Summary card**: `sm:p-6`

## Accessibility

- Child selector uses native `<select>` (accessible by default) with associated `<label>`
- Summary card icons are decorative: `aria-hidden="true"` on SVGs; meaning conveyed by text
- Table uses semantic `<table>`, `<thead>`, `<tbody>`, `<th scope="col">`
- Status badges have sufficient color contrast (green-800 on green-100; red-800 on red-100)
- Color is never the sole indicator:
  - At-risk rows: red background **plus** status badge text
  - Low frequency: red text **plus** row highlight
  - Low média: semibold red text **plus** status badge
- Focus visible: `focus:ring-2 focus:ring-indigo-500 focus:outline-none` on all interactive elements
- Loading states announced: `aria-live="polite"` on loading container
